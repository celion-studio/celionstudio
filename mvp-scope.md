# MVP Scope — Digital Product Builder

작성일: 2026-04-14
버전: v0.1
문서 목적: PRD를 기반으로 MVP 범위를 P0/P1/P2로 나누고, 첫 출시에서 반드시 포함할 기능과 제외할 기능을 명확히 정리한다.

## 1. MVP 목표
사용자가 주제 또는 기존 콘텐츠를 입력하면, 10분 안에 배포 가능한 전자책/PDF 가이드 초안과 배포용 카피 패키지를 얻을 수 있어야 한다.

핵심 가치:
- 빠른 구조화
- 기존 콘텐츠 재가공
- 배포 가능한 패키지 생성

## 2. 출시 전략
첫 버전은 “전자책/PDF 가이드 생성 + 배포 패키지”에 집중한다.

의도적으로 하지 않을 것:
- 복잡한 자동화 연동
- 고급 협업 기능
- 디자인 툴 수준의 편집기
- 결제/CRM/DM 자동화 자체 구현

즉, 첫 버전은 “발행 준비 완료 상태를 만들어주는 툴”이다.

---

## 3. 범위 기준
기능을 아래 세 기준으로 판단한다.

1. 첫 가치 경험에 직접 기여하는가?
2. 개발 복잡도 대비 사용자 가치가 높은가?
3. 기존 툴과 경쟁하기보다 보완하는 방향인가?

## 4. P0 — 반드시 포함
이 기능이 없으면 MVP가 성립하지 않는다.

### P0-1. 프로젝트 생성
설명:
새 디지털상품 프로젝트를 생성할 수 있어야 한다.

포함:
- 새 프로젝트 만들기
- 프로젝트 제목 임시 저장
- 생성 방식 선택: 새로 만들기 / 기존 콘텐츠 재가공

왜 필요한가:
모든 작업의 시작점이자 저장 단위가 필요하다.

### P0-2. 상품 브리프 입력
설명:
사용자가 최소한의 정보로 상품 방향을 정의할 수 있어야 한다.

포함:
- 상품 타입: 전자책 / PDF 가이드
- 주제
- 타겟 독자
- 목적
- 톤
- 분량

제외:
- 업종별 고급 옵션
- 복잡한 브랜드 설정

### P0-3. 기존 콘텐츠 붙여넣기
설명:
사용자가 기존 글, 스크립트, 메모를 붙여넣어 재가공할 수 있어야 한다.

포함:
- 텍스트 입력 영역
- 다중 소스 텍스트 붙여넣기 허용
- 요약/중복 제거/재구성 입력으로 활용

제외:
- URL 자동 크롤링
- 파일 업로드 파서
- 유튜브 자동 transcript 수집

판단:
초기에는 입력을 단순화하고, 자동 수집보다 수동 붙여넣기로 검증한다.

### P0-4. 제목/부제/목차 생성
설명:
사용자가 구조를 선택할 수 있도록 AI가 제안해야 한다.

포함:
- 제목 후보 5개
- 부제 후보 3개
- 목차 3안
- 상품 약속 문장

포함 이유:
대부분 사용자는 글보다 구조에서 먼저 막힌다.

### P0-5. 구조 선택 및 수정
설명:
사용자가 제안된 구조를 선택하고 가볍게 수정할 수 있어야 한다.

포함:
- 제목 선택
- 목차안 선택
- 챕터 추가/삭제
- 챕터명 수정

제외:
- 복잡한 drag-and-drop 고급 인터랙션

### P0-6. 본문 초안 생성
설명:
선택된 구조를 바탕으로 서문, 챕터 본문, 요약, CTA를 생성해야 한다.

포함:
- 서문
- 3~7개 챕터
- 요약/체크리스트 중 하나
- CTA

성공 조건:
사용자가 “이건 바로 다듬어서 배포할 수 있겠다”는 수준의 초안을 봐야 한다.

### P0-7. 기본 편집
설명:
생성된 초안을 최소한 수정할 수 있어야 한다.

포함:
- 텍스트 직접 수정
- 챕터 단위 재생성
- 길이 조정
- 톤 조정

제외:
- 실시간 공동 편집
- 고급 버전 히스토리

### P0-8. 배포 패키지 생성
설명:
문서 초안뿐 아니라 실제 배포에 필요한 카피를 함께 생성해야 한다.

포함:
- 상품 소개문
- 랜딩페이지용 짧은 설명
- SNS 홍보 문구 3개
- 댓글 유도 문구 3개
- DM 첫 문구

제외:
- 채널별 고급 자동화 시퀀스
- 이메일 자동 발송 연동

이 기능의 중요성:
이 제품이 단순 AI 글쓰기 툴과 달라지는 가장 핵심 포인트다.

### P0-9. PDF export
설명:
사용자가 결과물을 파일로 저장하고 배포할 수 있어야 한다.

포함:
- PDF export
- 텍스트 export

제외:
- 정교한 레이아웃 커스터마이징
- 템플릿 디자인 엔진

### P0-10. 저장 및 재진입
설명:
사용자가 나중에 다시 돌아와 수정할 수 있어야 한다.

포함:
- 브리프 저장
- 구조 저장
- 본문 저장
- 배포 패키지 저장
- 최근 프로젝트 목록

## 5. P1 — 출시 직후 붙이면 강한 기능
핵심은 아니지만, 제품 매력을 크게 높인다.

### P1-1. 체크리스트형 상품 템플릿
- 전자책 외에 체크리스트형 구조 제공

### P1-2. 여러 콘텐츠 소스 조합 UX 개선
- 입력 블록 분리
- 소스별 라벨링
- 어떤 소스를 얼마나 반영했는지 요약

### P1-3. CTA 타입 선택
- 뉴스레터 구독 유도
- 상담 신청 유도
- 유료 상품 연결 유도

### P1-4. 이메일 opt-in 문구 생성
- 구독 폼 문구
- 전달 메일 초안
- thank-you 문구

### P1-5. 표지 스타일 선택
- 심플 / 크리에이터형 / 프로페셔널형 등 기본 스타일

### P1-6. 간단한 공유 링크
- 결과물을 링크 형태로 공유 가능
- 단순 뷰 페이지 수준

## 10. 반드시 검증해야 할 가설
### 가설 1
사용자는 0부터 쓰는 기능보다 기존 콘텐츠 재가공 기능에 더 높은 가치를 느낀다.

### 가설 2
배포 패키지 생성은 단순 본문 생성보다 차별화 포인트로 작동한다.

### 가설 3
전자책/PDF 가이드 하나의 포맷만으로도 초기 유료 전환 테스트가 가능하다.

### 가설 4
복잡한 자동화 연동 없이도 “복사해서 바로 쓰는 배포 카피”만으로 충분한 효용을 준다.

## 11. 첫 출시 기준 (Release Gate)
아래가 모두 충족되면 MVP 출시 가능으로 본다.

- 사용자가 새 프로젝트를 만들 수 있다
- 브리프 입력 후 구조 제안을 받을 수 있다
- 제목/목차를 선택하고 본문 초안을 만들 수 있다
- 최소한의 편집이 가능하다
- 소개문/홍보문구/DM 문구를 생성할 수 있다
- PDF export가 안정적으로 된다
- 저장 후 재진입이 가능하다

---

## 12. 출시 후 관찰 포인트
- 어떤 진입점이 더 많이 쓰이는가: 새로 만들기 vs 기존 콘텐츠 재가공
- 사용자가 가장 많이 수정하는 구간은 어디인가: 제목 / 목차 / 본문 / CTA
- 배포 패키지 중 실제로 가장 많이 복사되는 것은 무엇인가
- PDF export까지 가는 데 가장 큰 이탈 구간은 어디인가
