# Wireframe — Digital Product Builder

작성일: 2026-04-14
버전: v0.1
문서 목적: PRD를 바탕으로 MVP 화면 구조와 핵심 인터랙션을 텍스트 와이어프레임 형태로 정리한다.

## 1. 설계 원칙
- 사용자는 10분 안에 첫 결과물을 봐야 한다
- 생성 흐름은 복잡한 설정보다 빠른 진행을 우선한다
- 구조를 먼저 결정하고 본문을 생성한다
- 마지막 화면은 항상 배포 액션으로 끝난다
- 문서 편집기가 아니라 “상품 조립기”처럼 느껴져야 한다

## 2. 전체 정보 구조
1. Dashboard
2. New Product / Brief Setup
3. Structure Builder
4. Draft Editor
5. Distribution Pack
6. Export / Share Modal

---

## 3. Screen 1 — Dashboard
목적:
- 새 상품 만들기 시작
- 기존 작업 이어하기
- 주요 템플릿 진입

텍스트 와이어프레임:

```text
+--------------------------------------------------------------+
| LOGO                    [검색]                  [프로필/설정] |
+--------------------------------------------------------------+
|                                                              |
|  디지털상품을 가장 빠르게 발행하세요                          |
|  기존 콘텐츠나 아이디어를 전자책/PDF 가이드로 바꿔보세요      |
|                                                              |
|  [새 디지털상품 만들기]   [기존 콘텐츠로 만들기]              |
|                                                              |
+--------------------------------------------------------------+
| 최근 작업                                                     |
|  - 인스타 성장 가이드 v2                [이어하기] [복제]     |
|  - 코치용 리드마그넷 초안               [이어하기] [복제]     |
|  - 뉴스레터 → PDF 재가공                [이어하기] [복제]     |
+--------------------------------------------------------------+
| 추천 템플릿                                                   |
|  [리드마그넷형] [유료 판매형] [초보자 가이드형] [워크북형]   |
+--------------------------------------------------------------+
| 하단 보조 영역                                                |
|  - 최근 export 파일                                           |
|  - 배포 가이드 팁                                             |
+--------------------------------------------------------------+
```

핵심 요소:
- Primary CTA: 새 디지털상품 만들기
- Secondary CTA: 기존 콘텐츠로 만들기
- 최근 작업 리스트
- 템플릿 빠른 선택

상호작용 포인트:
- 최근 작업 이어하기
- 최근 작업 복제하기
- 템플릿을 누르면 Brief Setup 화면으로 프리필 이동

## 4. Screen 2 — New Product / Brief Setup
목적:
- 상품 방향 정의
- 최소 입력으로 생성 시작

텍스트 와이어프레임:

```text
+--------------------------------------------------------------+
| Step 1 of 4 — 상품 브리프 설정                               |
+--------------------------------------------------------------+
| 상품 타입                                                    |
|  ( ) 전자책   ( ) PDF 가이드   ( ) 체크리스트형 가이드       |
|                                                              |
| 생성 방식                                                    |
|  ( ) 새로 만들기   ( ) 기존 콘텐츠 재가공                    |
|                                                              |
| 주제                                                         |
|  [ 인스타 크리에이터를 위한 첫 리드마그넷 만들기        ]    |
|                                                              |
| 타겟 독자                                                    |
|  [ 팔로워는 있지만 이메일 리스트가 없는 초보 크리에이터 ]    |
|                                                              |
| 목적                                                         |
|  [무료 리드수집 v]                                           |
|                                                              |
| 톤                                                           |
|  [실전형 v]                                                  |
|                                                              |
| 분량                                                         |
|  [보통 v]                                                    |
|                                                              |
| 참고 콘텐츠 / 메모                                            |
|  [ 텍스트 붙여넣기 영역                                      ]|
|  [ URL 추가 ] [파일 붙여넣기(후순위) ]                       |
|                                                              |
|                                    [취소] [다음: 구조 생성]  |
+--------------------------------------------------------------+
```

보조 UX:
- 우측 또는 하단에 “이 입력으로 어떤 결과가 나오나요?” 미리보기
- 예시 placeholder 제공
- 필수 입력이 비어 있어도 최소 모드로 진행 가능

핵심 검증:
- 사용자는 3분 이내 입력을 끝낼 수 있어야 한다

## 5. Screen 3 — Structure Builder
목적:
- 제목, 부제, 목차, 포지셔닝을 결정
- 사용자가 상품의 뼈대를 먼저 확정

레이아웃:
- 좌측: AI 제안 후보들
- 중앙: 선택된 구조 미리보기
- 우측: 상품 포지셔닝 패널

텍스트 와이어프레임:

```text
+----------------------+---------------------------+----------------------+
| 제목 후보            | 선택된 구조               | 상품 포지셔닝        |
|                      |                           |                      |
| 1. ...               | 제목: 인스타...           | 타겟 독자            |
| 2. ...               | 부제: ...                 | 초보 크리에이터      |
| 3. ...               |                           |                      |
| 4. ...               | 목차                      | 상품 약속            |
| 5. ...               | 1장 문제 정의             | 팔로워를 리드로      |
|                      | 2장 흔한 실수             | 전환하는 첫 PDF      |
| 부제 후보            | 3장 구조 만들기           |                      |
| - ...                | 4장 CTA 설계              | 목적                 |
| - ...                | 5장 배포하기              | 무료 리드수집        |
| - ...                |                           |                      |
| 목차안 1/2/3         | [챕터 추가] [챕터 삭제]   | 톤 / 분량            |
| [안1 선택]           | [텍스트 편집]             | 실전형 / 보통        |
| [안2 선택]           |                           |                      |
| [안3 선택]           |                           |                      |
+----------------------+---------------------------+----------------------+
| [이전]                           [구조 다시 제안] [본문 생성] |
+----------------------------------------------------------------+
```

핵심 액션:
- 제목 선택
- 목차 선택
- 챕터 순서 수정
- 상품 약속 문구 확인
- 본문 생성 시작

필수 UX 포인트:
- “구조가 마음에 안 들면 다시 생성”이 쉬워야 함
- 하나의 목차만 보여주지 말고 반드시 3안을 보여줘야 함

## 6. Screen 4 — Draft Editor
목적:
- 챕터별 본문 생성 및 수정
- 단순 장문 편집이 아니라 상품 품질을 끌어올리는 보조 액션 제공

레이아웃:
- 좌측: 챕터 네비게이션
- 중앙: 본문 에디터
- 우측: AI 액션 패널

텍스트 와이어프레임:

```text
+------------------+----------------------------------+---------------------+
| 챕터 목록         | 본문 에디터                       | AI 보조 패널         |
|                  |                                  |                     |
| 표지             | [제목]                           | 톤                  |
| 서문             | [부제]                           | [친절] [전문]       |
| 1장 문제 정의    |                                  | [실전] [크리에이터] |
| 2장 흔한 실수    | 서문 본문...                     |                     |
| 3장 구조 만들기  |                                  | 길이                |
| 4장 CTA 설계     | 1장 본문...                      | [짧게] [보통] [길게]|
| 5장 배포하기     |                                  |                     |
| 요약             | 2장 본문...                      | 보강 액션            |
| CTA              |                                  | [예시 추가]         |
|                  | 3장 본문...                      | [체크리스트 추가]    |
|                  |                                  | [더 구체적으로]      |
|                  |                                  | [재생성]            |
+------------------+----------------------------------+---------------------+
| 하단 액션: [자동 저장됨] [미리보기] [배포 패키지 생성]        |
+----------------------------------------------------------------+
```

핵심 기능:
- 챕터 단위 편집
- 챕터 단위 재생성
- 톤 변경
- 길이 조정
- 예시 추가
- 체크리스트 추가
- CTA 영역 편집

필수 UX 포인트:
- 사용자가 “AI가 내 글을 전부 덮어쓸까” 불안하지 않도록 챕터 단위 조작
- 자동 저장 상태 노출
- 최종 CTA를 별도 섹션으로 강조

## 7. Screen 5 — Distribution Pack
목적:
- 생성된 상품을 실제로 배포하기 위한 자산을 준비

레이아웃:
- 상단: 상품 요약
- 중단: 배포 자산 탭
- 하단: export/share 액션

텍스트 와이어프레임:

```text
+--------------------------------------------------------------+
| Step 4 of 4 — 배포 패키지                                    |
+--------------------------------------------------------------+
| 상품명: 인스타 크리에이터를 위한 첫 리드마그넷               |
| 형식: 전자책 / 목적: 무료 리드수집 / 상태: 배포 준비 완료    |
+--------------------------------------------------------------+
| 탭: [소개문] [랜딩 문구] [SNS 홍보] [댓글 유도] [DM] [이메일]|
+--------------------------------------------------------------+
| 소개문                                                       |
|  이 가이드는 ...                                             |
|  [복사] [다시 생성]                                           |
|                                                              |
| 랜딩 문구                                                    |
|  헤드라인: ...                                               |
|  서브카피: ...                                               |
|  CTA: ...                                                    |
|  [복사] [다시 생성]                                           |
|                                                              |
| SNS 홍보 문구                                                |
|  1. ...                                                      |
|  2. ...                                                      |
|  3. ...                                                      |
|  [전체 복사]                                                  |
+--------------------------------------------------------------+
| [PDF 내보내기] [텍스트 export] [공유 링크 생성] [저장]       |
+--------------------------------------------------------------+
```

핵심 포인트:
- 배포용 문구를 유형별로 바로 복사 가능하게 제공
- export 버튼을 가장 강하게 노출
- 사용자가 “이제 뭘 해야 하지?”가 아니라 “이제 바로 배포하면 되네”를 느껴야 함

## 8. Export / Share Modal
목적:
- 실제 배포 직전 최종 출력과 링크 확보

텍스트 와이어프레임:

```text
+---------------------------------------------------+
| 내보내기                                          |
+---------------------------------------------------+
| 파일 형식                                         |
|  ( ) PDF                                          |
|  ( ) 텍스트 문서                                  |
|                                                   |
| 제목                                               |
|  [ 인스타 크리에이터를 위한 첫 리드마그넷 ]       |
|                                                   |
| 설명                                               |
|  [ 짧은 설명 입력 ]                               |
|                                                   |
| 공유 옵션                                          |
|  [ ] 공유 링크 생성                               |
|  [ ] 다운로드 링크 생성                           |
|                                                   |
|                              [취소] [생성하기]    |
+---------------------------------------------------+
```

출력 후 상태:
- 파일 다운로드 완료
- 공유 링크 복사 버튼 제공
- “이제 홍보 문구를 사용해 배포하세요” 보조 가이드 표시

## 9. 핵심 컴포넌트 목록
- Header / Global Navigation
- Recent Projects List
- Template Cards
- Brief Form
- Title Candidate List
- TOC Candidate Cards
- Positioning Summary Card
- Chapter Sidebar
- Rich Text Draft Area
- AI Action Panel
- Distribution Tabset
- Export Modal

## 10. MVP에서 반드시 필요한 상태값
- draft_status: not_started / generating / ready / edited
- export_status: not_started / generating / ready / failed
- content_source_type: fresh / repurposed / mixed
- distribution_assets_status: not_started / ready
- autosave_status: saving / saved / error

## 11. 화면별 핵심 KPI 연결
- Dashboard → 새 프로젝트 시작률
- Brief Setup → 구조 생성 진입률
- Structure Builder → 본문 생성 전환율
- Draft Editor → 배포 패키지 생성 전환율
- Distribution Pack → PDF export 및 공유 링크 생성률

## 12. 디자인 톤 가이드
- 복잡한 문서 편집기보다 명확한 카드/패널 중심 구조
- “쓰기”보다 “조립”과 “발행”이 느껴지는 CTA 문구 사용
- 생성 과정은 진입장벽 없이 간단해야 하지만, 결과는 전문적으로 보여야 함
