# PRD — Digital Product Builder

작성일: 2026-04-14
버전: v0.1
문서 목적: 크리에이터/전문가/1인사업자가 기존 콘텐츠나 아이디어를 빠르게 디지털상품으로 만들고 바로 배포할 수 있게 해주는 서비스의 MVP 제품 요구사항 정의

## 1. 제품 한 줄 정의
콘텐츠를 디지털상품으로 바꾸고, 배포 가능한 패키지까지 한 번에 만들어주는 발행 툴.

## 2. 배경
많은 크리에이터, 강사, 코치, 컨설턴트, 1인사업자는 전자책/PDF 가이드/체크리스트 같은 디지털상품을 만들고 싶어 한다. 하지만 실제로는 다음 문제에서 막힌다.

- 무엇을 상품으로 만들지 정리가 안 됨
- 목차와 구조를 못 잡음
- 긴 글을 빠르게 쓰기 어려움
- 기존 콘텐츠를 상품형 구조로 재가공하기 어려움
- PDF만 만들고 끝나서 실제 배포까지 이어지지 않음
- 랜딩 문구, 소개문, 홍보 문구를 별도로 또 만들어야 함

현재 AI 툴은 대체로 글 초안 생성에 머물거나, 반대로 DM 자동화/이메일 자동화 등 배포 채널 쪽에 치우쳐 있다. 사용자가 실제로 원하는 것은 “파일 하나 생성”이 아니라 “배포 가능한 디지털상품 패키지”다.

## 3. 제품 비전
사용자가 이미 가진 콘텐츠, 경험, 지식, 메모를 빠르게 구조화해 디지털상품으로 만들고, 바로 배포할 수 있는 상태까지 만드는 것이 목표다.

장기적으로는 전자책 생성기가 아니라 “크리에이터용 디지털상품 발행 엔진”으로 확장한다.

## 4. 목표
### 4.1 비즈니스 목표
- 크리에이터/전문가 대상의 명확한 문제 해결형 SaaS 포지셔닝 확보
- 무료 체험에서 첫 상품 생성 경험까지의 시간을 10분 이내로 단축
- “단순 AI 글쓰기 툴”이 아닌 “상품화 + 배포 툴”로 인식시키기
- 구독형 수익모델이 가능한 제품 구조 확보

### 4.2 사용자 목표
- 내 콘텐츠를 빠르게 디지털상품으로 바꾸고 싶다
- 처음부터 다 쓰지 않고, 기존 글이나 영상 스크립트를 활용하고 싶다
- 전자책/PDF 파일뿐 아니라 소개문/홍보문구/배포문구까지 한 번에 받고 싶다
- 최소한의 입력으로 바로 초안을 얻고 싶다

## 5. 대상 사용자
### 핵심 ICP
1. 크리에이터
- 인스타그램, 유튜브, 뉴스레터, 블로그, X 등에서 콘텐츠를 발행하는 사람
- 콘텐츠를 리드마그넷 또는 유료 상품으로 전환하고 싶은 사람

2. 코치/강사/컨설턴트
- 전문성을 PDF 가이드, 전자책, 워크북으로 포장하려는 사람
- 상담/강의/서비스 판매를 돕는 리드 수집 자산이 필요한 사람

3. 1인사업자/정보형 브랜드 운영자
- 고객 교육자료, 리드 수집 자료, 세일즈 보조 자료가 필요한 사람

### 제외 범위
- 장편 출판용 원고 중심의 저자 툴
- 전문 편집/출판 워크플로우가 필요한 출판사 중심 솔루션
- 고도화된 마케팅 자동화 엔진 자체를 대체하는 제품

## 6. 핵심 문제 정의
사용자는 “글을 쓰는 것”보다 “상품처럼 보이게 구조화하고 배포하는 것”에서 더 크게 막힌다.

핵심 문제는 5단계로 요약된다.
1. 주제 정리
2. 구조화
3. 초안 생성
4. 상품 포장
5. 배포 준비

MVP는 이 다섯 단계를 하나의 흐름으로 연결해야 한다.

## 7. 제품 원칙
1. 적게 입력하고 빠르게 결과를 본다
2. 새로 쓰기보다 기존 콘텐츠 재가공에 강해야 한다
3. 문서 생성이 아니라 상품 생성처럼 느껴져야 한다
4. 마지막 단계는 항상 배포 액션이어야 한다
5. DM 자동화 툴과 경쟁하지 않고, 그 앞단의 상품/카피/패키지 생성에 집중한다

## 8. 핵심 가치 제안
- 블로그 글, 강의안, 영상 스크립트, 메모를 디지털상품으로 바꿔준다
- 제목, 목차, 본문뿐 아니라 소개문과 홍보 문구까지 같이 만든다
- PDF만 만드는 것이 아니라 배포 가능한 패키지를 제공한다
- 크리에이터의 기존 콘텐츠를 재활용해 상품화 시간을 크게 줄인다

## 9. MVP 범위
MVP는 전자책(ebook) 생성 중심이다.
내부 구조는 향후 체크리스트, 워크북, 템플릿 번들 등으로 확장 가능하게 설계한다.

### 9.1 포함 범위
- 주제 기반 디지털상품 생성
- 기존 콘텐츠 기반 재구성
- 제목/부제/목차 생성
- 본문 초안 생성
- CTA 및 마무리 페이지 생성
- 배포용 패키지 문구 생성
- PDF export
- 공유용 링크 또는 다운로드 링크 생성

### 9.2 제외 범위
- ManyChat 직접 연동
- 인스타/메타 댓글 감지 자동화
- Gumroad/Stripe 등 고도화 결제 연동
- 팀 협업/권한 관리
- 고급 디자인 커스터마이징
- 성과 분석 대시보드

## 10. 핵심 사용자 시나리오
### 시나리오 A: 새 주제로 전자책 만들기
1. 사용자가 새 디지털상품 만들기를 시작한다
2. 상품 타입, 주제, 타겟 독자, 목적, 톤, 분량을 입력한다
3. 시스템이 제목/부제/목차 안을 제안한다
4. 사용자가 구조를 선택하거나 수정한다
5. 시스템이 본문 초안을 생성한다
6. 사용자가 일부 챕터를 수정한다
7. 시스템이 소개문/랜딩 문구/SNS 홍보 문구/DM 문구를 생성한다
8. 사용자가 PDF로 내보내고 공유 링크를 얻는다

### 시나리오 B: 기존 콘텐츠를 재가공해 상품 만들기
1. 사용자가 블로그 글, 유튜브 스크립트, 메모 등을 붙여넣는다
2. 시스템이 핵심 내용을 요약하고 중복을 제거한다
3. 시스템이 전자책 구조와 상품 포지셔닝을 제안한다
4. 사용자가 제목과 목차를 선택한다
5. 시스템이 재구성된 본문 초안을 만든다
6. 시스템이 배포용 패키지까지 함께 생성한다
7. 사용자가 PDF export 후 바로 배포한다

## 11. 기능 요구사항
### FR-1. 상품 생성 브리프 입력
사용자는 아래 항목을 입력할 수 있어야 한다.
- 상품 타입: 전자책 / PDF 가이드 / 체크리스트형 가이드
- 생성 방식: 새로 만들기 / 기존 콘텐츠 재가공
- 주제
- 타겟 독자
- 목적: 무료 리드수집 / 유료 판매 / 팬 배포 / 교육용
- 톤: 친절한 / 전문적인 / 실전형 / 크리에이터형
- 분량: 짧게 / 보통 / 길게

성공 기준:
- 입력 3분 이내에 완료 가능해야 함
- 필수 입력만으로도 결과 생성 가능해야 함

### FR-2. 기존 콘텐츠 가져오기
사용자는 여러 형태의 기존 콘텐츠를 입력할 수 있어야 한다.
- 텍스트 붙여넣기
- 블로그 글 원문
- 유튜브 transcript
- 강의안/메모 초안
- 여러 콘텐츠 조합

시스템은 다음을 수행해야 한다.
- 핵심 내용 추출
- 중복 제거
- 논리 흐름 정리
- 챕터화 가능한 구조 제안

### FR-3. 제목/부제/목차 생성
시스템은 아래를 제안해야 한다.
- 제목 후보 5개
- 부제 후보 3개
- 목차 3안
- 타겟 독자 정의 문장
- 상품의 약속 문장 1개

사용자는 제안안을 선택 또는 수정할 수 있어야 한다.

### FR-4. 본문 초안 생성
시스템은 선택된 구조를 바탕으로 초안을 생성해야 한다.
최소 산출물:
- 표지 제목
- 서문
- 3~7개 챕터
- 각 챕터 본문
- 요약 또는 체크리스트
- CTA

사용자는 챕터 단위로 다음을 할 수 있어야 한다.
- 재생성
- 길이 조절
- 톤 조정
- 예시 추가
- 체크리스트 추가

### FR-5. 배포 패키지 생성
시스템은 아래 자산을 함께 생성해야 한다.
- 상품 소개문
- 랜딩페이지용 짧은 설명
- SNS 홍보 문구 3개
- 댓글 유도 문구 3개
- DM 전달용 첫 문구
- 이메일 opt-in 문구
- 다운로드 페이지 문구

### FR-6. 내보내기 및 공유
사용자는 아래 작업을 할 수 있어야 한다.
- PDF export
- 텍스트 문서 export
- 공유 링크 생성
- 다운로드 링크 생성

### FR-7. 저장 및 재편집
사용자는 작업 중인 디지털상품을 저장하고 나중에 다시 수정할 수 있어야 한다.
최소 저장 항목:
- 브리프
- 선택한 제목/목차
- 챕터 초안
- 배포 패키지

## 12. 화면 요구사항
### 화면 1. 대시보드
목적:
- 새 상품 생성 시작
- 기존 작업 이어하기

구성:
- 새 디지털상품 만들기 버튼
- 기존 콘텐츠로 만들기 버튼
- 최근 작업 목록
- 템플릿 진입점

### 화면 2. 생성 시작 화면
목적:
- 상품 방향 정의

구성:
- 상품 타입 선택
- 생성 방식 선택
- 주제 입력
- 타겟 독자 입력
- 목적 선택
- 톤 선택
- 분량 선택
- 참고 콘텐츠 입력 영역

### 화면 3. 구조 설계 화면
목적:
- 제목/목차/포지셔닝 결정

구성:
- 제목 후보 리스트
- 부제 후보 리스트
- 목차 3안
- 상품 약속 문장
- 선택/수정 인터랙션

### 화면 4. 본문 생성 및 편집 화면
목적:
- 챕터별 초안 생성 및 수정

구성:
- 챕터 사이드바
- 본문 편집 영역
- 톤/길이 조정 버튼
- 예시/체크리스트 추가 기능
- CTA 편집 영역

### 화면 5. 배포 패키지 화면
목적:
- 배포용 자산 생성 및 export

구성:
- 상품 소개문
- 랜딩 문구
- SNS 홍보 문구
- 댓글 유도 문구
- DM/이메일 전달 문구
- PDF export 버튼
- 공유 링크 생성 버튼

## 13. 사용자 플로우
### 기본 플로우
대시보드
→ 생성 시작
→ 구조 설계
→ 본문 생성/편집
→ 배포 패키지
→ export/share

### 핵심 UX 원칙
- 첫 결과물은 10분 안에 보여준다
- 긴 설정 과정 없이 빠르게 생성에 들어간다
- 사용자가 구조를 먼저 결정하고 글은 나중에 생성한다
- 마지막 액션은 저장이 아니라 배포 준비여야 한다

- 본문 초안 생성

### P2

## 15. 성공 지표
### 핵심 지표
- 첫 상품 생성 완료율
- 생성 시작 후 첫 결과물 도달 시간
- PDF export 완료율
- 생성 후 배포 패키지 사용률
- 무료 → 유료 전환율

### 초기 목표 예시
- 신규 사용자 중 60% 이상이 첫 구조 생성까지 도달
- 구조 생성 사용자 중 40% 이상이 PDF export까지 도달
- 유료 전환 사용자의 다수가 기존 콘텐츠 재가공 기능을 사용

## 16. 비기능 요구사항
- 생성 속도: 주요 단계별 응답은 사용자가 흐름을 끊기지 않게 유지해야 함
- 저장 안정성: 초안 손실이 없어야 함
- export 안정성: PDF 출력 오류가 적어야 함
- 사용성: 비전문 사용자도 설명 없이 흐름을 따라갈 수 있어야 함

## 17. 경쟁/대안 대비 포지션
### 사용자가 현재 대체하는 방법
- ChatGPT로 본문 생성
- Notion/Docs에서 수동 편집
- Canva에서 PDF 디자인
- ManyChat/Zapier로 배포 자동화 연결
- Gumroad/노션/링크 페이지로 판매 또는 전달

### 차별점
- 글 초안 생성에서 끝나지 않음
- 기존 콘텐츠 재가공이 핵심 흐름에 포함됨
- 구조화 → 본문 → 배포 패키지가 하나의 제품 안에 연결됨
- “파일 생성”이 아니라 “상품 발행 준비”를 제공함

## 18. 수익모델
### Free
- 월 1~2개 생성 제한
- 일부 export 제한
- 템플릿 제한
- 워터마크 포함 가능

### Pro
- 핵심 타겟 요금제
- 월 생성량 확대
- 기존 콘텐츠 재가공 포함
- 배포 패키지 전체 제공
- 워터마크 제거

### Creator / Business
- 여러 프로젝트 관리
- 브랜드 톤 저장
- 향후 연동 기능 포함
- 고급 템플릿 제공

### 추가 수익원
- 생성 크레딧 업셀
- 업종별 템플릿 판매
- AI 초안 + 사람 편집 서비스
